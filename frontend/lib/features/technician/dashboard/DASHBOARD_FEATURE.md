# Technician Dashboard Feature

> **Backend contract**: `GET /api/technicians/dashboard/`
> **Presentation entry-point**: `TechnicianDashboardScreen`
> **State provider**: `technicianDashboardProvider` (generated by Riverpod)

---

## 1. Domain Entities

All entities are `@freezed` immutable value objects. They carry no database
annotations and know nothing about JSON.

### `TechnicianDashboardEntity`
The root entity for the entire screen.

| Field | Type | Nullable | Description |
|---|---|---|---|
| `walletBalance` | `double` | no | Current virtual wallet balance in Rs. Patched in-place by realtime events — never triggers a full screen reload. |
| `isOnline` | `bool` | no | Whether the technician is currently available for matchmaking. Optimistically flipped by `setOnline()`. |
| `profilePicture` | `String?` | yes | Absolute CDN URL for the avatar. `null` → fallback icon in the header. |
| `upNextJob` | `UpNextJobEntity?` | yes | The single next scheduled job. `null` → empty-state card shown. |
| `laterTodayJobs` | `List<LaterTodayJobEntity>` | no | Remaining jobs after `upNextJob`. Empty list → "No more jobs" text. |
| `metrics` | `DashboardMetricsEntity` | no | Today's summary stats. |

### `UpNextJobEntity`

| Field | Type | Description |
|---|---|---|
| `jobId` | `int` | Primary key — used for deep-linking to the job detail screen. |
| `serviceTitle` | `String` | Human-readable service name (e.g. "AC Deep Wash"). |
| `scheduledTime` | `DateTime` | Used for time display and the live countdown chip. |
| `customerName` | `String` | Displayed in the job card meta row. |
| `addressText` | `String` | Human-readable address displayed below customer name. |
| `lat` / `lng` | `double` | Map centre coordinates for `JobLocationMap`. Also used to build the Google Maps navigation URL. |

### `LaterTodayJobEntity`

| Field | Type | Description |
|---|---|---|
| `jobId` | `int` | Primary key for future deep-link. |
| `serviceTitle` | `String` | Rendered in the list row title. |
| `scheduledTime` | `DateTime` | Formatted with `DateFormat.jm()` for the time column. |
| `addressText` | `String` | Secondary line in the list row. |

### `DashboardMetricsEntity`

| Field | Type | Description |
|---|---|---|
| `jobsCompletedToday` | `int` | Displayed as a plain integer in the Jobs Completed card. |
| `cashCollectedToday` | `double` | Formatted with `NumberFormat('#,##0')` → "Rs. 3,500". |

## 2. Sealed Failure Hierarchy

```
Exception
└── TechnicianDashboardFailure (sealed, abstract)
    ├── DashboardNetworkFailure   — no internet + no cache
    ├── DashboardServerFailure    — backend 5xx / unreachable
    ├── DashboardPermissionFailure — 403: user is not a technician
    └── DashboardParsingFailure   — unexpected JSON contract
```

All subclasses carry a `message: String` field used verbatim in the UI error
state. `DashboardServerFailure` is the only one where the message is dynamic
(comes from the backend); the others have default messages.

---

## 3. Repository Interface

```dart
abstract class TechnicianDashboardRepository {
  /// Throws [DashboardNetworkFailure]   — no internet and no cache.
  /// Throws [DashboardServerFailure]    — backend error.
  /// Throws [DashboardPermissionFailure]— user is not a technician (403).
  /// Throws [DashboardParsingFailure]   — JSON contract mismatch.
  Future<TechnicianDashboardEntity> getDashboard();
}
```

## 4. Data Models (JSON ↔ Entity)

Plain Dart classes (not Freezed). Each model owns `fromJson`, `toJson`, and
`toEntity()`. The `toJson` output is what the `LocalDataSource` persists to
`SharedPreferences`.

### `TechnicianDashboardModel.fromJson` — expected JSON contract

```json
{
  "wallet_balance": 1500.00,
  "is_online": true,
  "profile_picture": "https://cdn.example.com/pic.jpg",
  "up_next_job": {
```

## 5. Data Sources

### `TechnicianDashboardRemoteDataSource`
- Makes a single `GET /api/technicians/dashboard/` with the JWT from
  `FlutterSecureStorage`.
- On 200: deserialises via `TechnicianDashboardModel.fromJson`.
- On non-200: throws `HttpFailure(code, message)` which the repository maps.

### `TechnicianDashboardLocalDataSource`
- Backed by `SharedPreferences` under the key
  `"technician_dashboard_cache"`.
- `cache(TechnicianDashboardModel)`: serialises to JSON string and writes.
- `getCached()`: reads and deserialises. Returns `null` if no cache exists.

## 6. Repository Impl — Offline-First Flow

```
getDashboard()
  ├── try: RemoteDataSource.getDashboard()
  │     ├── success → cache result → return entity
  │     └── HttpFailure → map to domain failure → rethrow
  └── on SocketException:
        ├── LocalDataSource.getCached() != null → return cached entity
        └── cache empty → throw DashboardNetworkFailure
```

The `_mapFailures` helper switches on `HttpFailure.code`:

| HTTP Code | Domain Failure |
|---|---|
| `403` / `"permission_denied"` | `DashboardPermissionFailure` |
| `5xx` / `"server_error"` | `DashboardServerFailure(message)` |
| parse exception | `DashboardParsingFailure` |
| other | `DashboardServerFailure(message)` |

---

## 7. Error Propagation Pipeline

The 4-step pipeline is strictly followed — no raw JSON ever reaches the UI.

```
Step 1 — RemoteDataSource
  Non-200 response → throw HttpFailure(code, message, errors)

Step 2 — RepositoryImpl (_mapFailures)
  switch (HttpFailure.code)
    "permission_denied" → throw DashboardPermissionFailure()
    "server_error"      → throw DashboardServerFailure(message)
    _                   → throw DashboardServerFailure(message)
  SocketException       → check cache → DashboardNetworkFailure | cached entity

Step 3 — Domain
  sealed class TechnicianDashboardFailure implements Exception
  Subclasses carry the user-facing message string.

Step 4 — UI (TechnicianDashboardScreen._ErrorState)
  switch (error)
    DashboardNetworkFailure()     → "No internet connection. Pull down to retry."
    DashboardPermissionFailure()  → "You do not have permission..."
    DashboardParsingFailure()     → "Could not read dashboard data. Please retry."
    DashboardServerFailure(msg)   → msg (verbatim from backend)
    _                             → "Something went wrong. Please retry."
```

## 8. Dependency Injection Wiring

All providers live in `presentation/providers/dependency_injection.dart` and are
generated by `riverpod_generator`.

```
sharedPreferencesProvider          (from onboarding feature, reused)
    └── TechnicianDashboardLocalDataSource

technicianDashboardHttpClientProvider  (keepAlive: true)
    └── TechnicianDashboardRemoteDataSource

technicianDashboardRepositoryProvider
    ├── TechnicianDashboardRemoteDataSource
    └── TechnicianDashboardLocalDataSource

technicianDashboardProvider            (auto-dispose async notifier)
    └── TechnicianDashboardRepository
```

The HTTP client is `keepAlive: true` so it is not torn down between navigations.
Its `onDispose` calls `client.close()` when the app exits.

## 9. Presentation Layer

### State — `TechnicianDashboardState`

```dart
@freezed
class TechnicianDashboardState {
  TechnicianDashboardEntity dashboard;
  @Default(AsyncValue.data(null)) AsyncValue<void> toggleStatus;
}
```

`toggleStatus` is a **sub-`AsyncValue`** deliberately separated from the
screen-level `AsyncValue`. This means:
- A failed toggle shows a SnackBar without wiping the entire dashboard.
- A successful toggle shows no spinner on the full screen — only the toggle chip
  animates.

### Notifier — `TechnicianDashboardNotifier`

| Method | Trigger | Behaviour |
|---|---|---|
| `build()` | Screen mount | Fetches via repository; sets screen-level `AsyncValue`. |
| `refresh()` | Pull-to-refresh, high-urgency realtime event | Full re-fetch; transitions screen through `AsyncLoading`. |
| `setOnline(bool)` | Online toggle tap | Optimistic flip of `isOnline`; sets `toggleStatus: AsyncLoading`. On stub success → `AsyncData(null)`. On failure → rolls back `isOnline`, sets `toggleStatus: AsyncError`. No-op if `desired == current` or dashboard not loaded. |
| `onWalletBalanceEvent(double)` | Realtime `WALLET_BALANCE_UPDATED` | Single-field patch — no `AsyncLoading` on screen. |
| `onForcedOfflineEvent()` | Realtime forced-offline event | Sets `isOnline: false` without a screen reload. Idempotent when already offline. |

```
TechnicianDashboardScreen  (ConsumerWidget)
```

### Reusable Map Widget — `JobLocationMap`

Located at `lib/core/widgets/map/job_location_map.dart`.

**Frozen interface** (do not change when migrating to Google Maps):
```dart
JobLocationMap({
  required double lat,
  required double lng,
  double zoom = 15.0,
  double height = 140.0,
  BorderRadius borderRadius = …,
})
```

Currently backed by `flutter_map` + OpenStreetMap tiles.
`IgnorePointer` wraps the entire map so all scroll/drag events pass through to
the parent `CustomScrollView` — the technician cannot accidentally interact with
the map.

**To migrate to Google Maps SDK:** replace the body of `job_location_map.dart`
only. No other file needs to change.
